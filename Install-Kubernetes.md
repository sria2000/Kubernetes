# Kubernetes Cluster Installation (Master & Worker Nodes)

This guide explains how to build a Kubernetes cluster using **kubeadm** on Ubuntu.

**Environment**

- Ubuntu 24.xx (or supported version)
- 1 Master Node
- 1 or more Worker Nodes
- Container Runtime: containerd
- Kubernetes Version: v1.35.x

---

# Prerequisites

- Install VMware Fusion / VMware Workstation (or VirtualBox)
- Create two Ubuntu virtual machines
- Configure the network as **NAT**
- Ensure:
  - Both machines can reach the Internet
  - Both machines can SSH to each other
  - Static IP addresses are configured

# Common Configuration (Run on ALL Nodes)

Run all commands as **root**.

## 1. Update Ubuntu

```bash
apt update
apt upgrade -y
```

---

## 2. Disable Swap

Kubernetes requires swap to be disabled.

```bash
sed -i '/swap/d' /etc/fstab
swapoff -a
```

Verify:

```bash
free -h
```

---

## 3. Disable Firewall

```bash
systemctl disable --now ufw
```

Verify:

```bash
systemctl status ufw
```

---

## 4. Load Required Kernel Modules

Create the configuration file.

```bash
cat >/etc/modules-load.d/containerd.conf <<EOF
overlay
br_netfilter
EOF
```

Load the modules.

```bash
modprobe overlay
modprobe br_netfilter
```

Verify:

```bash
lsmod | egrep "overlay|br_netfilter"
```

---

## 5. Configure Kernel Parameters

```bash
cat >/etc/sysctl.d/kubernetes.conf <<EOF
net.bridge.bridge-nf-call-ip6tables = 1
net.bridge.bridge-nf-call-iptables  = 1
net.ipv4.ip_forward                 = 1
EOF
```

Apply:

```bash
sysctl --system
```

Verify:

```bash
sysctl net.ipv4.ip_forward
```

---

## 6. Install Kubernetes Repository

Install required packages.

```bash
apt-get update
apt-get install -y apt-transport-https ca-certificates curl gpg
```

Create keyring directory.

```bash
install -p -m 0755 -d /etc/apt/keyrings
```

Download repository key.

```bash
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.35/deb/Release.key \
| gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```

```bash
chmod a+r /etc/apt/keyrings/kubernetes-apt-keyring.gpg
```

Add the Kubernetes repository.

```bash
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.35/deb/ /' \
> /etc/apt/sources.list.d/kubernetes.list
```

---

## 7. Install Kubernetes Components

```bash
apt-get update
apt-get install -y kubelet kubeadm kubectl
```

Prevent automatic upgrades.

```bash
apt-mark hold kubelet kubeadm kubectl
```

Enable kubelet.

```bash
systemctl enable --now kubelet
```

Verify:

```bash
kubeadm version
kubectl version --client
```

---

## 8. Install containerd

```bash
apt update
apt upgrade -y

apt install -y containerd

mkdir -p /etc/containerd

containerd config default > /etc/containerd/config.toml
```

Enable Systemd Cgroup.

```bash
sed -i 's/SystemdCgroup = false/SystemdCgroup = true/' /etc/containerd/config.toml
```

Restart containerd.

```bash
systemctl enable containerd
systemctl restart containerd
```

Verify:

```bash
systemctl status containerd
```

---

## 9. Install net-tools

```bash
apt install -y net-tools
```

---

## 10. Enable SSH Password Authentication

```bash
sed -i 's/^PasswordAuthentication.*/PasswordAuthentication yes/' /etc/ssh/sshd_config

echo "PermitRootLogin yes" >> /etc/ssh/sshd_config

systemctl reload sshd
```

---

## 11. Configure Hosts File

Example:

```bash
cat >> /etc/hosts <<EOF
192.168.68.105 k8s-master master
192.168.68.113 k8-node01 worker1
EOF
```

Verify:

```bash
ping k8s-master
ping worker1
```

# Master Node Configuration

Pull Kubernetes images.

```bash
kubeadm config images pull
```

Initialize the cluster.

Replace the IP address with your Master Node IP.

```bash
kubeadm init \
--apiserver-advertise-address=192.168.68.105 \
--pod-network-cidr=192.168.0.0/16
```

**IMPORTANT**

Save the output of the following command generated by `kubeadm init`.

```bash
kubeadm join ...
```

This will be required on every worker node.

---

## Configure kubectl

For root:

```bash
export KUBECONFIG=/etc/kubernetes/admin.conf
```

Or for a normal user:

```bash
mkdir -p $HOME/.kube

cp /etc/kubernetes/admin.conf $HOME/.kube/config

chown $(id -u):$(id -g) $HOME/.kube/config
```

---

## Enable kubectl Auto-completion

```bash
source <(kubectl completion bash)

echo "source <(kubectl completion bash)" >> ~/.bashrc

alias k=kubectl

complete -o default -F __start_kubectl k
```

# Join Worker Node

Run the command generated by the Master.

Example:

```bash
kubeadm join 192.168.68.105:6443 \
--token <TOKEN> \
--discovery-token-ca-cert-hash sha256:<HASH>
```

# Verify Cluster

On the Master Node:

```bash
kubectl get nodes
```

Expected output:

```text
NAME          STATUS   ROLES           AGE
k8s-master    Ready    control-plane
k8-node01     Ready    <none>
```

# Generate a New Join Token

If the original token expires:

```bash
kubeadm token create --print-join-command
```

List existing tokens.

```bash
kubeadm token list
```

If required:

```bash

| openssl rsa -pubin -outform der 2>/dev/null \
| openssl dgst -sha256
```
